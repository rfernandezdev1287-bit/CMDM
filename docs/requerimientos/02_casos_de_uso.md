# 🏛️ CMDM: Casos de Uso del Sistema

## 1. Definición de Actores
*   **Operador (Tú):** El Juez y Mediador soberano desde el dispositivo móvil (Chrome).
*   **Búnker Integrado:** El organismo compuesto por **Anti-Gravity** (Editor/Orquestador), **MCP** (Protocolo) y **OLAma** (Ejecutor local).
*   **Arquitecto (Tú / Gemini):** La IA de alto nivel para estrategia y doctrina.
*   **CMDM:** La herramienta que permite el flujo, la voz y la memoria.

## 2. Escenarios de Uso (Flujo Maestro)

### CU-01: El Espejo de Orquestación
El Operador abre la app y ve en tiempo real el diálogo entre Anti-Gravity y OLAma. Puede ver qué archivos se están tocando y qué plan está ejecutando el Búnker.

### CU-02: El Ciclo de Intercesión Humana
1. El Operador ve una respuesta del Búnker, la copia y la pega en el canal del Arquitecto.
2. El Operador añade instrucciones extra (vía teclado o **voz**) y envía.
3. El Arquitecto responde.
4. El Operador copia la respuesta del Arquitecto, la pega en el canal del Búnker, añade precisiones y envía para ejecución.

### CU-03: Compresión Cavernícola Socrática
Al transferir texto entre IAs, el Operador activa el "Squeeze". El sistema elimina paja verbal pero ancla las preguntas socráticas para asegurar que la IA deduzca la solución.

### CU-04: Snapshot de Engrama
Si la sesión se interrumpe, la CMDM ofrece un botón de "Restaurar" para volver al punto exacto de la conversación, cargando el SSD y el último Engrama del proyecto.

### CU-05: Dictado de Instrucciones (Voz a Texto)
1. El Operador pulsa el botón de micrófono en el móvil.
2. Chrome activa el reconocimiento local; el Operador dicta la instrucción técnica.
3. El texto se inserta en el input maestro. El Operador lo valida y lo envía.

### CU-06: Audición de Dictámenes (Texto a Voz)
1. El Operador pulsa el botón de "Escuchar" en una burbuja de chat (Búnker o Arquitecto).
2. El sistema lee en voz alta el contenido, permitiendo que el Operador analice la respuesta sin necesidad de mirar la pantalla.
